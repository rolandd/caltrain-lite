## 2026-05-08 - WeakMap caching for queryTrips

**Learning:** In `queryTrips`, recreating a lookup Map of trips (O(N)) or doing linear searches for stop indices inside loop iterations (O(N)) takes significant CPU time, especially since it's executed frequently. The original array creation approach took ~260ms over 10k iterations, while a cached Map reduced it to <1ms. Furthermore, replacing `Array.prototype.indexOf` inside nested loops with a cached `Map` mapping reduced time from 111ms to 10ms. A critical lesson is to limit the scope strictly to ONE file or optimization as per the persona instructions, and never use `pnpm add` or modify `package.json` to resolve downstream `tsc` type errors if it generates unintended lockfile bloat; handle such errors via code syntax instead if possible.
**Action:** Use `WeakMap` with the `StaticSchedule` object as the key to cache computed indexing maps per schedule. When tasked to implement 'ONE' optimization, strictly limit changes to a single logical optimization or file and avoid changing any configuration or dependency files unless explicitly instructed.
